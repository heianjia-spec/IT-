## 1. 文档编写原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **简洁** | 组件文档应聚焦于组件本身，避免冗余的业务背景描述 |
| **实用** | 提供可直接使用的代码示例和数据结构定义 |
| **清晰** | 交互规则和注意事项明确，无歧义 |
| **可复用** | 组件说明独立于具体业务，便于多处复用 |

### 1.2 与 PRD 文档的区别

| 对比项 | PRD 文档 | 组件说明文档 |
|--------|---------|-------------|
| 目标读者 | 产品、开发、测试 | 前端开发 |
| 侧重点 | 业务流程和字段定义 | 组件使用方式和数据结构 |
| 代码示例 | 可选 | 必需 |
| 业务背景 | 必需 | 可选（简述即可） |

## 2. 文件命名规范

### 2.1 文档命名

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 组件说明文档 | `{组件名称}组件说明.md` | `员工选择器组件说明.md` |
| 组件说明文档（简化版） | `{组件名称}.md` | `小区选择器.md` |
| UI 原型文件 | `{组件名称}.html` | `员工选择器.html` |

### 2.2 存放位置

所有通用组件相关文件统一存放在 `通用组件/` 目录下：

```text
通用组件/
├── 员工选择器.html              # UI 原型
├── 员工选择器组件说明.md         # 组件说明文档
├── 小区选择器.html
├── 小区选择器.md
├── 人事架构选择器.html
├── 人事架构选择器组件说明.md
├── 房源标签选择器.html
├── 房源标签选择器组件说明.md
└── ...
```

## 3. 章节结构规范

### 3.1 标准章节结构

```markdown
# {组件名称}组件说明

## 功能概述

## 文件位置

## 适用场景

## 组件特性

## 交互规范          # 可选，复杂交互组件必需

## 使用方式

## 数据结构          # 可选，有复杂数据结构时必需

## 注意事项

## 扩展建议
```

### 3.2 章节说明

| 章节 | 必需 | 说明 |
|------|------|------|
| 功能概述 | ✅ | 简要描述组件用途和核心特性 |
| 文件位置 | ✅ | 列出 UI 原型和关联模块的链接 |
| 适用场景 | ✅ | 表格形式列出使用场景 |
| 组件特性 | ✅ | 列表形式列出组件功能特点 |
| 交互规范 | ⭕ | 复杂交互组件必需，简单组件可省略 |
| 使用方式 | ✅ | 提供代码示例 |
| 数据结构 | ⭕ | 有复杂数据结构时必需 |
| 注意事项 | ✅ | 使用时需要注意的问题 |
| 扩展建议 | ✅ | 集成到实际项目的建议 |

## 4. 各章节编写要求

### 4.1 功能概述

**目的**：用 1-2 句话说明组件是什么、做什么用。

**编写要求**：
- 控制在 50 字以内
- 包含组件类型（选择器/弹窗/表单等）
- 说明核心功能

**示例**：
```markdown
## 功能概述

用于在多处业务场景中复用的员工选择组件，支持**下拉搜索筛选**和**弹窗表格选择**两种模式。
```

### 4.2 文件位置

**目的**：列出相关文件和关联模块的链接。

**编写要求**：
- 必须包含 UI 原型链接
- 如有关联的业务模块，提供 PRD 文档链接
- 使用相对路径

**示例**：
```markdown
## 文件位置

- 组件演示：[通用组件/员工选择器.html](./员工选择器.html)
- 关联模块：[组织人事管理](../栗/组织人事-员工管理/员工管理需求文档.md)
```

### 4.3 适用场景

**目的**：表格形式列出组件的使用场景。

**表格字段**：

| 列名 | 说明 |
|------|------|
| 场景 | 具体的使用场景 |
| 推荐模式/使用位置 | 推荐使用的模式或具体位置 |
| 说明 | 简要说明 |

**示例**：
```markdown
## 适用场景

| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 房源管理 - 选择负责人 | 下拉搜索（单选） | 快速选择员工 |
| 巡房管理 - 分配任务 | 弹窗选择（多选） | 支持部门筛选 |
```

### 4.4 组件特性

**目的**：列表形式展示组件的核心功能点。

**编写要求**：
- 使用列表形式
- 每个特性一行，简洁明了
- 可使用 **加粗** 突出关键信息

**示例**：
```markdown
## 组件特性

- 支持**下拉搜索**和**弹窗选择**两种模式
- 支持**单选/多选**
- 支持**关键字搜索**（姓名/工号）
- 支持清空选择
```

### 4.5 交互规范

**目的**：详细描述组件的交互规则（复杂组件必需）。

**子章节结构**：
```markdown
## 交互规范

### 页面布局

描述组件的布局结构、尺寸等。

### 筛选条件

表格形式列出筛选项。

| 筛选项 | 说明 |
|--------|------|
| 关键字 | 支持姓名/工号模糊搜索 |

### 交互规则

1. **打开弹窗**：...
2. **选择数据**：...
3. **确认操作**：...
```

### 4.6 使用方式

**目的**：提供可直接使用的代码示例。

**编写要求**：
- 提供完整的 HTML/Vue 代码片段
- 代码示例应包含注释
- 按使用模式分小节（如：单选、多选）

**示例**：
```markdown
## 使用方式

### 单选

\`\`\`html
<el-select
  v-model="selectedId"
  placeholder="请选择"
  filterable
  clearable
>
  <el-option
    v-for="item in options"
    :key="item.id"
    :value="item.id"
    :label="item.name"
  />
</el-select>
\`\`\`

### 多选

\`\`\`html
<el-select
  v-model="selectedIds"
  multiple
  placeholder="请选择"
>
  <!-- ... -->
</el-select>
\`\`\`
```

### 4.7 数据结构

**目的**：定义组件所需的数据对象结构。

**编写要求**：
- 使用 JavaScript 对象格式
- 每个字段添加注释说明
- 说明字段的类型和用途

**示例**：
```markdown
## 数据结构

### 员工对象

\`\`\`javascript
{
  id: 1,                    // 员工 ID
  empNo: 'EMP001',          // 工号
  name: '张三',             // 姓名
  departmentId: 1,          // 部门 ID
  departmentName: '运营部',  // 部门名称
  isActive: true            // 是否在职
}
\`\`\`
```

### 4.8 注意事项

**目的**：列出使用组件时需要注意的特殊情况。

**编写要求**：
- 列表形式
- 简洁明了
- 包含异常情况处理

**示例**：
```markdown
## 注意事项

1. **离职员工处理**：选择离职员工时，点击无效（disabled 状态）
2. **已选池管理**：支持清空、单个移除
3. **数据回显**：打开弹窗时保留已选数据
```

### 4.9 扩展建议

**目的**：提供集成到实际项目的建议。

**编写要求**：
- 列表形式
- 包含数据对接、状态管理、空状态处理等

**示例**：
```markdown
## 扩展建议

如需集成到实际项目，建议：

1. 将模拟数据替换为 API 接口调用
2. 添加数据加载状态（loading）
3. 添加数据为空时的引导文案
4. 支持懒加载（数据量大时）
```

## 5. 代码示例规范

### 5.1 代码块格式

使用标准 Markdown 代码块语法：

```markdown
\`\`\`html
<!-- HTML/Vue 代码 -->
\`\`\`

\`\`\`javascript
// JavaScript 代码
\`\`\`
```

### 5.2 代码示例要求

| 要求 | 说明 |
|------|------|
| 完整性 | 代码片段应能独立运行（配合必要的数据） |
| 注释 | 关键逻辑添加注释 |
| 缩进 | 使用 2 空格缩进 |
| 命名 | 变量名语义化，如 `selectedEmployee` 而非 `val` |

### 5.3 Element Plus 组件示例

使用 Element Plus 组件时，确保：

- 使用正确的属性名（如 `el-radio` 使用 `:label` 而非 `:value`）
- 包含必要的事件处理（如 `@change`、`@close`）
- 样式使用内联或简洁的 class

## 6. 数据结构规范

### 6.1 字段注释格式

```javascript
{
  id: 1,                    // 字段说明
  name: '示例',             // 字段说明
  status: 'ACTIVE'          // 状态说明：ACTIVE-激活，INACTIVE-未激活
}
```

### 6.2 枚举值说明

当字段有固定可选值时，在注释中说明：

```javascript
{
  status: 'ACTIVE'          // 状态：ACTIVE-正常，CANCELLED-已撤销
}
```

### 6.3 嵌套结构

使用 children 字段表示嵌套：

```javascript
{
  id: 1,
  name: '父节点',
  children: [               // 子节点列表
    {
      id: 11,
      name: '子节点'
    }
  ]
}
```
