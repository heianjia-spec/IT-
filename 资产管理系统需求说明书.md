## 1. 引言

### 1.1 项目背景

为方便企业对 IT 资产进行统一管理，需开发一套 IT 资产管理系统，对企业内部的电脑、打印机、服务器、各类 IT 耗材以及少量软件等进行全生命周期管理，实现资产的可追溯、可统计、可告警。

### 1.2 项目目标

- 建立 IT 资产统一台账，覆盖硬件、软件、IT 耗材、备件、云资源等类别。
- 提供资产的录入、导入、导出、变更、报废等全流程管理能力。
- 通过仪表盘与报表，向管理者呈现资产总量、分布与状态。
- 通过提醒与告警机制，确保资产异动、到期、保修等问题能够及时被发现和处理。
- 支持与 ITSM 系统对接，为 CMDB 提供基础数据。

### 1.3 名词术语

| 术语 | 说明 |
| --- | --- |
| 资产 | 企业拥有或租用的、用于 IT 运行的硬件、软件、耗材等 |
| 台账 | 资产的完整属性记录 |
| NC 编号 | 企业内部对资产的业务编号 |
| 在用 | 资产当前正在被使用 |
| 在库 | 资产已入库但未被使用 |
| 报废 | 资产已退出使用并完成报废处理 |
| ITSM | IT 服务管理系统 |
| CMDB | 配置管理数据库 |

## 2. 总体功能结构

系统按业务划分为以下五大模块，外加一项外部集成能力：

1. 仪表盘
2. 资产管理
3. 基础数据
4. 通知与告警
5. 系统设置
6. CMDB 集成（与 ITSM 对接）

模块之间的关系：

- 基础数据为资产管理提供分类、部门、位置、供应商等基础字典。
- 资产管理是系统的核心业务模块。
- 仪表盘基于资产管理的数据进行统计与展示。
- 通知与告警基于资产状态变化与到期数据进行触发。
- 系统设置负责用户的账号与权限管理。
- CMDB 集成负责把本系统数据对接到 ITSM 系统。

## 3. 功能需求

### 3.1 仪表盘

#### 3.1.1 资产概览

**功能描述：** 在仪表盘首页展示企业 IT 资产的总体统计信息。

**展示指标：**

- 资产总数
- 在用量
- 闲置量（在库量）
- 报废量

**展示形式：** 数字卡片 + 图表（饼图/柱状图等）。

#### 3.1.2 分类报表

**功能描述：** 按维度对资产进行分类统计。

**统计维度：**

- 按类别（硬件/软件/IT 耗材/备件/云资源等）
- 按部门
- 按状态（在用/在库/报废等）

#### 3.1.3 到期提醒

**功能描述：** 在仪表盘上展示即将到期的资产清单。

**提醒类型：**

- 维保到期提醒
- 合同到期提醒
- 折旧到期提醒

#### 3.1.4 自定义报表

**功能描述：** 用户可自定义报表的统计维度与展示字段，生成自定义报表。

---

### 3.2 资产管理

资产管理模块包含 5 个子功能：资产列表、新增资产、导入资产、导出资产、自定义报表。

#### 3.2.1 资产列表

##### 3.2.1.1 资产台账

**功能描述：** 提供资产数据的列表化展示，支持对资产的增、删、改、查。

**主要功能：**

- 支持按资产编号、名称、NC 编号、类别、状态、部门、责任人等条件检索。
- 支持新增、编辑、删除资产记录。
- 支持查看资产详情。

**资产台账基础字段：**

| 字段 | 说明 |
| --- | --- |
| 资产编号 | 系统唯一标识 |
| 资产名称 | 资产的业务名称 |
| NC 编号 | 企业内部业务编号 |
| 类别 | 硬件 / 软件 / IT 耗材 / 备件 / 云资源等 |
| 规格型号 | 资产的型号描述 |
| SN 码 / 序列号 | 厂商提供的序列号 |
| 资产状态 | 在用 / 在库 / 报废等 |
| 备注 | 其他补充说明 |

##### 3.2.1.2 资产分类

**功能描述：** 支持对资产按业务类别进行分类管理。

**分类示例：**

- 硬件：服务器、PC、网络设备、打印机等
- 软件：许可证、订阅
- IT 耗材
- 备件
- 云资源

##### 3.2.1.3 资产标签 / 二维码

**功能描述：** 支持为资产生成可打印的标签，标签中包含资产二维码。

**功能要点：**

- 可自定义标签样式（字段、尺寸等）。
- 通过扫码可快速查询资产的台账信息和维护记录。

##### 3.2.1.4 资产导入导出

**功能描述：** 支持 Excel / CSV 格式的批量导入与导出。

> 详见 3.2.3 与 3.2.4。

#### 3.2.2 新增资产

##### 3.2.2.1 功能描述

实现单条资产的录入功能。根据资产类别不同，系统自动加载对应的属性录入表单。

录入表单分为四个信息区块：

1. 基础信息
2. 归属信息
3. 采购信息
4. 硬件信息

##### 3.2.2.2 基础信息

| 字段 | 说明 |
| --- | --- |
| 资产编号 | 系统唯一标识 |
| 资产名称 | 资产的业务名称 |
| NC 编号 | 企业内部业务编号 |
| 硬件分类 | 例如：打印机、服务器、PC 等 |
| 品牌 | 厂商品牌 |
| 序列号 | SN 码 |
| 资产状态 | 在用 / 在库 / 报废等 |
| 规格型号 | 资产的型号描述 |
| 备注 | 补充说明 |

##### 3.2.2.3 归属信息

| 字段 | 说明 |
| --- | --- |
| 使用部门 | 资产归属的使用部门 |
| 责任人 | 资产的责任人 |

**状态联动规则：**

- 当【资产状态】选择"在用"时，显示以下字段：
  - 机房名称
  - 机柜编号
- 当【资产状态】选择"在库"时，显示以下字段：
  - 存放位置

##### 3.2.2.4 采购信息

| 字段 | 说明 |
| --- | --- |
| 购置日期 | 资产购入日期 |
| 生产日期 | 资产出厂日期 |
| 购置金额 | 采购价格 |
| 折旧年限 | 用于计算折旧 |
| 供应商 | 采购的供应方 |
| 保修截止日期 | 保修到期的日期 |
| 保修状态 | 在保 / 过保 等 |

##### 3.2.2.5 硬件信息（按硬件分类区分）

不同硬件分类对应不同的硬件信息字段。以下为两种典型示例：

**示例一：硬件-打印机**

| 字段 | 说明 |
| --- | --- |
| 端口数量 | 打印机可用端口数 |
| 管理地址 | 管理 IP 地址 |
| MAC 地址 | 网络接口 MAC |
| 固件版本 | 打印机固件版本号 |

**示例二：硬件-服务器**

| 字段 | 说明 |
| --- | --- |
| CPU | 处理器型号与配置 |
| 内存信息 | 内存容量与规格 |
| 盘位信息 | 硬盘盘位数与已用情况 |
| 管理地址 | 管理 IP 地址（如 iDRAC / iLO） |
| 网卡信息 | 网卡数量与配置 |

> 说明：其他硬件分类（如 PC、网络设备等）的硬件信息字段应按设备类型进行扩展，扩展字段应在基础数据中维护。

#### 3.2.3 导入资产

**功能描述：** 支持通过 Excel 表批量导入资产信息。

**功能要点：**

- 系统提供标准导入样表（模板），用户按样表格式填写后上传。
- 导入时按所选硬件分类，校验对应字段。
- 导入失败时，向用户反馈失败行号与原因。

#### 3.2.4 导出资产

**功能描述：** 用户可自定义需要导出的属性字段，按所选字段导出资产信息。

**功能要点：**

- 支持按检索结果导出。
- 支持选择导出的字段列。
- 导出格式为 Excel / CSV。

#### 3.2.5 自定义报表

**功能描述：** 用户可自定义报表的统计维度、筛选条件和展示字段，生成自定义报表。

**功能要点：**

- 支持按类别、部门、状态、时间等维度组合统计。
- 支持选择展示字段。
- 报表可导出。

### 3.3 基础数据

基础数据为系统其他模块提供基础字典维护能力，包含以下子模块：

#### 3.3.1 资产分类

**功能描述：** 维护资产的类别字典，例如硬件（服务器、PC、网络设备、打印机等）、软件、IT 耗材、备件、云资源等。

#### 3.3.2 部门管理

**功能描述：** 维护企业内部部门信息，用于资产归属与统计。

#### 3.3.3 位置管理

**功能描述：** 维护资产存放位置信息，包括机房、机柜、库房位置等。

#### 3.3.4 供应商

**功能描述：** 维护供应商档案，用于资产采购信息关联。

---

### 3.4 通知与告警

#### 3.4.1 邮件配置

**功能描述：** 可配置邮件服务器参数，实现自动告警邮件发送。

#### 3.4.2 消息通知

**功能描述：** 系统按事件类型，向相关用户发送通知。

**通知类型：**

- 审批通知
- 到期提醒（维保到期、合同到期、折旧到期等）
- 操作通知

#### 3.4.3 异常告警

**功能描述：** 当资产发生异常事件时，系统自动进行告警。

**告警类型：**

- 资产新增告警
- 资产异动告警（如归属变更、状态变更等）
- 盘点差异告警

---

### 3.5 系统设置

#### 3.5.1 用户管理

**功能描述：** 维护系统使用者的账号、信息与权限。

**功能要点：**

- 用户密码设置（修改密码、重置密码）。
- 用户信息设置（账号、姓名、部门、联系方式等）。
- 用户启用 / 停用。

---

### 3.6 CMDB 集成

**功能描述：** 与 IT 服务管理（ITSM）系统对接，向 CMDB 提供资产基础数据。

**集成方向：** 本系统作为资产数据源，向 ITSM / CMDB 输出资产配置信息。

## 4. 业务规则汇总

| 编号 | 规则描述 |
| --- | --- |
| BR-01 | 新增资产时，资产编号在系统内唯一，不可重复。 |
| BR-02 | 资产的硬件分类决定了新增资产表单中"硬件信息"区块显示的字段集合。 |
| BR-03 | 资产【资产状态】为"在用"时，归属信息必须填写机房名称与机柜编号。 |
| BR-04 | 资产【资产状态】为"在库"时，归属信息必须填写存放位置。 |
| BR-05 | 资产保修截止日期早于当前日期时，保修状态自动标识为"过保"。 |
| BR-06 | 维保到期、合同到期、折旧到期事件触发仪表盘到期提醒与消息通知。 |
| BR-07 | 资产新增、归属变更、状态变更、盘点差异等事件触发异常告警。 |
| BR-08 | 资产导入必须使用系统提供的标准样表，否则导入失败。 |
| BR-09 | 资产导出时，导出字段以用户自定义选择为准。 |
| BR-10 | 资产标签 / 二维码中的二维码需可被扫码识别并跳转至该资产的详情。 |
